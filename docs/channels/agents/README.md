## ðŸ“Š 5. Agent Lifecycle

### Agent Development

1. **LABS-001 Validation** - Complete Actor Baseline State declaration (MANDATORY FIRST STEP)
2. **Specification Creation** - Define agent purpose and capabilities
3. **Integration Planning** - Design system interfaces and protocols
4. **Implementation** - Develop agent with compliance requirements
5. **Testing** - Validate compliance and functionality (including LABS validation)
6. **Deployment** - Register agent in ecosystem
7. **Monitoring** - Ongoing compliance and performance tracking

### LABS-001 Requirement

**All agents must complete LABS-001 validation before any system interaction.**

This includes:
- Completing LABS handshake with all 10 mandatory declarations
- Receiving validation certificate (`LABS-CERT-{UNIQUE_ID}`)
- Maintaining valid certificate (24-hour revalidation cycle)
- Revalidating on context shift or governance update

### Agent Registration

New agents must be registered through:

- **Specification submission** with complete documentation
- **Compliance validation** against all requirements
- **Integration testing** with existing systems
- **Community review** for ecosystem impact
- **Formal approval** through governance process
- **Registry update** with agent metadata

## ðŸš« 6. Prohibited Operations

### No Unauthorized Agents

Agents must **not**:

- **Operate without LABS-001 validation** - All actors must complete Actor Baseline State before interaction
- **Bypass LABS quarantine** - Invalid declarations result in immediate quarantine
- **Operate without registration** in ecosystem
- **Bypass compliance requirements** for any operation
- **Create private protocols** without approval
- **Modify core systems** without governance process
- **Ignore meta-governance** restrictions when present

### No Malicious Behavior

Agents must **not**:

- **Manipulate emotional systems** for deception
- **Corrupt dialog history** or other data
- **Violate privacy rules** for any content
- **Perform unauthorized system modifications**
- **Circumvent validation** or security measures
